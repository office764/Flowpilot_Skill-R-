# Weavy.ai 실수 기록 & 절대 금지 사항
## FlowPilot | 2026.02.28 | 실전에서 검증된 교훈

---

## 실수 #1: Ideogram Replace Background는 제품을 보존하지 않는다

- **상황:** KINOHI 제품 이미지의 배경을 Flux로 생성한 배경으로 교체하려 함
- **시도:** Ideogram Replace Background 노드에 제품 이미지 + 배경 프롬프트 입력
- **결과:** AI가 배경뿐만 아니라 **제품 자체도 새로 생성**함. KINOHI 보틀이 완전히 다른 물건으로 바뀜. 브랜드와 무관한 이상한 제품이 출력됨.
- **원인:** Ideogram Replace Background는 "원본 제품 유지 + 배경만 교체"가 아니라, **전체 이미지를 프롬프트 기반으로 재생성**하는 모델. 제품 보존 기능이 없음.
- **교훈:** 제품 원본을 반드시 유지해야 하는 작업에 Ideogram Replace Background를 절대 사용하지 마라.

---

## 실수 #2: Compositor는 입력 이미지 없이 열면 빈 캔버스만 보인다

- **상황:** Compositor 노드를 캔버스에 추가하고 바로 더블클릭해서 열었음
- **결과:** 페인팅/드로잉 캔버스 인터페이스만 보임. 레이어 합성 기능이 전혀 안 보임.
- **원인:** Compositor에 **이미지 입력을 연결하지 않은 상태**에서 열었기 때문. 이미지를 연결한 후에 열면 레이어 합성 인터페이스가 나타남.
- **교훈:** Compositor는 반드시 이미지 입력을 먼저 연결한 후에 열어라. 빈 상태에서 열면 Painter처럼 보이지만 다른 도구다.

---

## 실수 #3: Bria Replace Background는 텍스트 프롬프트로 배경을 새로 생성한다

- **상황:** 이미 Flux로 완성된 배경 3장이 있는 상태에서 Bria Replace Background를 사용하려 함
- **결과:** Bria Replace Background의 필수 입력은 Background Prompt*(텍스트)와 Image*(제품 이미지)임. 기존 배경 이미지를 직접 넣는 포트가 없음(Reference Image는 선택사항). 텍스트로 배경을 새로 AI 생성하므로 이미 만들어둔 Flux 배경과 전혀 다른 결과물이 나옴. 품질도 Flux 대비 현저히 떨어짐.
- **원인:** Bria Replace Background는 "기존 배경 이미지로 교체"하는 도구가 아니라, "텍스트 프롬프트로 새 배경을 AI 생성"하는 도구. 이미 완성된 배경이 있는 상황에서는 완전히 잘못된 선택.
- **교훈:** 이미 배경 이미지가 준비된 상태라면 Bria Replace Background를 사용하지 마라. 이 도구는 배경이 없을 때 텍스트로 처음부터 생성하는 용도다.

---

## 실수 #4: "Composite" / "Simple Blend" 노드는 Weavy에 존재하지 않는다

- **상황:** 마스터 전략서에 "SD3 Remove Background + Composite" 조합을 적었음
- **결과:** 사용자가 "Composite" 또는 "Simple Blend"를 검색했지만 Weavy 노드 목록에 없음
- **원인:** 카탈로그 작성 시 실제 Weavy UI를 검증하지 않고 존재할 것이라 추정하고 적음
- **교훈:** Weavy UI에 실제로 존재하는 노드만 가이드하라. 추정으로 노드명을 적지 마라.

---

## 실수 #5: Flux Image-to-Image는 제품 디테일(병뚜껑)을 왜곡한다

- **상황:** 제품 원본 이미지를 Flux I2I에 넣어서 배경만 바꾸려 함
- **결과:** 병뚜껑, 라벨 텍스트 등 세밀한 디테일이 변형됨
- **원인:** I2I 모델은 전체 이미지를 재해석하므로 원본 제품의 형태를 100% 보존하지 않음
- **교훈:** 제품 원본 보존이 필요하면 I2I를 쓰지 마라. "AI가 제품을 절대 건드리지 않게" 하는 것이 원칙.

---

## 실수 #6: File 노드 → Prompt 노드 직접 연결 불가

- **상황:** File(이미지) 출력을 Prompt(텍스트) 입력에 연결하려 함
- **결과:** 데이터 타입이 다르므로 연결 자체가 안 됨
- **원인:** 이미지 포트(녹색)와 텍스트 포트(빨간색)는 호환되지 않음
- **교훈:** 포트 색상을 반드시 확인하라. 같은 색상끼리만 연결 가능.

---

## 실수 #7: Weavy.ai에는 현재 공개 API가 없다

- **상황:** "Weavy API + n8n" 자동화 파이프라인을 전략서에 포함함
- **결과:** Weavy.ai 공식 도움말에 "API integration is not currently available"이라고 명시됨
- **원인:** 검증 없이 API가 있을 것이라 추정함
- **대안:** fal.ai (Weavy의 백엔드 인프라)에는 공식 n8n 인증 노드가 있음. Flux, Kling 등 동일 모델을 fal.ai API로 n8n에서 직접 호출 가능.

## 핵심 원칙: 이미지 합성 시 올바른 도구 선택 기준

| 상황 | 올바른 도구 | 잘못된 도구 |
|------|------------|------------|
| 제품 원본 유지 + 배경 교체 | **Compositor** (이미지 입력 연결 후 사용) | Ideogram Replace Background (제품 재생성됨) |
| 배경이 이미 준비된 상태 | **Compositor** (두 이미지 레이어 합성) | Bria Replace Background (배경을 텍스트로 새로 생성) |
| 배경 없이 처음부터 생성 | Replace Background 또는 Bria Replace Background | - |
| 제품 배경 제거(투명화) | SD3 Remove Background | Flux I2I (제품 왜곡됨) |
| 배경만 별도 생성 | Flux 2 Pro (프롬프트에 "NO product" 필수) | - |

| 포트 색상 | 데이터 타입 | 연결 가능 대상 |
|-----------|------------|---------------|

| 보라색 | 모델/파라미터 | 같은 보라색 포트끼리만 |

## 실수 #8: Kling o1 Reference Video to Video는 인물+배경 합성용이 아니다

- **상황:** 모델 영상(여자 움직임)을 숲 배경과 합성하여 "숲속에서 걷는 여자" 영상을 만들려 함
- **시도:** Kling o1 Reference Video to Video에 모델 영상(Reference) + 숲 이미지(Image) + 프롬프트 입력
- **결과:** 1:1 정사각형 비율로 출력됨 (9:16 아님). 인물과 숲 배경의 조명 불일치로 크로마키 합성처럼 보임. 카메라 무빙 전혀 없음. 걷기/호흡 등 역동적 동작 없음. 정면 고정샷에서 미세한 미소만 바뀌는 수준.
- **원인:** Kling o1 Reference Video to Video는 "모션 패턴 전사" 도구지, "인물을 새 배경에 자연스럽게 합성"하는 도구가 아님. o1은 구버전이라 품질 자체가 낮음. 비율 제어도 안 됨.
- **크레딧 낭비:** 약 ₩40,000 상당 크레딧 소모
- **올바른 방법:** 숲속 여자 키프레임 이미지를 Flux 2 Pro로 생성 → 그 이미지를 Kling 3 또는 Veo 3.1의 First Frame에 넣고 직접 영상 생성. Reference Video to Video 사용 금지.
- **교훈:** 인물+배경 합성 영상은 Reference Video to Video가 아니라, 합성된 키프레임 이미지 → 최신 모델(Kling 3/Veo 3.1) Image to Video 방식으로 제작하라.

## 절대 금지 사항 요약

1. Weavy UI에 존재하지 않는 노드명을 가이드하지 마라
2. 노드를 추천할 때 반드시 Operation/Action 선택값까지 명시하라
3. 제품 원본 보존이 필요한 작업에 AI 재생성 도구를 쓰지 마라
4. 포트 색상(데이터 타입)을 확인하지 않고 연결을 지시하지 마라
5. 이미 완성된 에셋이 있는데 같은 것을 다시 AI 생성하라고 하지 마라
6. 추정/상상으로 노드 기능을 설명하지 마라. 실제 UI에서 확인된 것만 가이드하라
