# B-Roll IF 노드 + 최종 결과 집계 — 간결 수정 가이드 v5.1
## FlowPilot | 2026.03.05

---

## 현재 상황 정리

스크린샷에서 확인한 사실:

- B-Roll 결과 조회에서 **URL이 정상적으로 들어오고 있다** (successFlag: 1, resultUrls에 MP4 URL 존재)
- 120초 대기면 Veo 3.1 영상이 **완성된다**
- **문제는 IF 노드의 TRUE/FALSE 출력이 아무 데도 연결되어 있지 않다** → 데이터가 여기서 끊긴다

---

## 수정할 것: 총 2가지

| 순서 | 뭘 하는가 | 소요 시간 |
|:---:|----------|:---------:|
| **1** | IF 노드의 TRUE 출력을 B-Roll 순차 루프에 연결 | 30초 |
| **2** | 최종 결과 집계 코드 교체 | 2분 |

---

## 수정 1: IF 노드 연결하기

### 지금 상태 (끊겨 있음)

```
대기 120초 → B-Roll 결과 조회 → IF → (TRUE) ???  아무것도 없음
                                    → (FALSE) ??? 아무것도 없음
```

### 해야 할 것

```
대기 120초 → B-Roll 결과 조회 → IF → (TRUE)  → B-Roll 순차 루프 (루프 반복)
                                    → (FALSE) → B-Roll 순차 루프 (루프 반복)
```

**TRUE든 FALSE든 둘 다 B-Roll 순차 루프로 돌아가야 한다.**

왜? SplitInBatches 루프는 "다음 item으로 넘어가라"는 신호가 와야 다음 B-Roll을 처리한다. 연결이 안 되어 있으면 루프가 멈춘다.

### 구체적 조작 방법

**Step 1-1: TRUE 출력 연결**

1. IF 노드의 오른쪽에 있는 **true** 포트 (위쪽 동그라미)에 마우스를 가져간다
2. 그 동그라미를 **클릭한 채로 드래그**한다
3. **B-Roll 순차 루프** 노드(SplitInBatches)의 **왼쪽 입력 포트**에 놓는다

**Step 1-2: FALSE 출력 연결**

1. IF 노드의 오른쪽에 있는 **false** 포트 (아래쪽 동그라미)에 마우스를 가져간다
2. 그 동그라미를 **클릭한 채로 드래그**한다
3. **똑같이** **B-Roll 순차 루프** 노드의 **왼쪽 입력 포트**에 놓는다

### 완성 후 연결도

```
B-Roll 순차 루프 ←──────────────────────────────────────┐
     ↓ (Port 1: 루프 진행)                               │
B-Roll 영상 생성 (Veo 3.1)                               │
     ↓                                                   │
대기 120초                                                │
     ↓                                                   │
B-Roll 결과 조회                                          │
     ↓                                                   │
IF (response 있는가?)                                     │
     ├── TRUE  ──────────────────────────────────────────┘
     └── FALSE ──────────────────────────────────────────┘

B-Roll 순차 루프 → (Port 0: 루프 완료) → 최종 결과 집계
```

## 수정 2: 최종 결과 집계 코드 전체 교체

### 왜 교체하는가

현재 코드가 `$input.all()`을 쓰는데, SplitInBatches의 "done" 포트가 누적된 잡다한 데이터를 보내서 talking_clips가 11개로 나온다.

`$input.all()` 대신 **`$('노드명').all()`로 각 노드의 출력을 직접 참조**하면 정확히 5개씩 나온다.

### 교체 방법

1. **최종 결과 집계** 노드를 더블클릭해서 연다
2. JavaScript 코드 영역을 클릭한다
3. **Ctrl+A** (전체 선택)
4. 아래 코드를 **Ctrl+V** (붙여넣기)

### 새 코드 (전체 — 복사용)

```javascript
// ============================================
// 최종 결과 집계 — v5.1 (2026.03.05)
// ============================================

// 1) 디렉터 AI 원본 5개 item (가장 믿을 수 있는 데이터)
const directorItems = $('Code in JavaScript1').all();

// 2) 대본 파싱에서 브랜드/제품 정보
const scriptData = $('대본 파싱').first().json;

// 3) B-Roll 결과 조회에서 영상 URL 가져오기
let brollPollItems = [];
try {
  brollPollItems = $('B-Roll 결과 조회').all();
} catch(e) {
  brollPollItems = [];
}

// 4) 토킹 클립 결과 조회에서 영상 URL 가져오기
//    ★ 아래 노드명이 실제 워크플로우와 다르면 에러남 → 정확한 이름으로 수정 필요
let talkingPollItems = [];
try {
  talkingPollItems = $('토킹 클립 결과 조회').all();
} catch(e) {
  try {
    talkingPollItems = $('InfiniteTalk 결과 조회').all();
  } catch(e2) {
    talkingPollItems = [];
  }
}

// ── URL 추출 함수 ──
function extractVideoUrl(pollData) {
  if (!pollData) return '';
  const d = pollData.json || pollData;
  try {
    // 경로 A: d.data.response.resultUrls[0]  ← 스크린샷에서 확인된 실제 구조
    if (d.data && d.data.response && d.data.response.resultUrls && d.data.response.resultUrls[0]) {
      return d.data.response.resultUrls[0];
    }
    // 경로 B: d.response.resultUrls[0]
    if (d.response && d.response.resultUrls && d.response.resultUrls[0]) {
      return d.response.resultUrls[0];
    }
    // 경로 C: d.data.response.resultUris[0]  (Uris 오타 대비)
    if (d.data && d.data.response && d.data.response.resultUris && d.data.response.resultUris[0]) {
      return d.data.response.resultUris[0];
    }
    // 경로 D: d.response.resultUris[0]
    if (d.response && d.response.resultUris && d.response.resultUris[0]) {
      return d.response.resultUris[0];
    }
    // 경로 E: resultJson 파싱
    const rj = d.data?.resultJson || d.resultJson || '';
    if (rj) {
      const parsed = typeof rj === 'string' ? JSON.parse(rj) : rj;
      return parsed.video?.url || parsed.url || parsed.output?.[0] || '';
    }
  } catch(e) {
    return '';
  }
  return '';
}

// ── 토킹 클립 5개 조립 ──
const talkingClips = [];
for (let i = 0; i < directorItems.length; i++) {
  const dir = directorItems[i].json;
  const videoUrl = extractVideoUrl(talkingPollItems[i]);
  talkingClips.push({
    segment_id: dir.timeline_id,
    section: dir.section,
    time_range: dir.time_range,
    tts_text: dir.tts_text,
    video_url: videoUrl,
    status: videoUrl ? 'completed' : 'missing'
  });
}

// ── B-Roll 클립 5개 조립 ──
const brollClips = [];
for (let i = 0; i < directorItems.length; i++) {
  const dir = directorItems[i].json;
  const videoUrl = extractVideoUrl(brollPollItems[i]);
  brollClips.push({
    broll_id: 'broll_' + (i + 1),
    broll_name: dir.broll_name || ('B-Roll ' + (i + 1)),
    section: dir.section,
    time_range: dir.time_range,
    video_url: videoUrl,
    status: videoUrl ? 'completed' : 'missing'
  });
}

// ── 집계 ──
const talkingOk = talkingClips.filter(c => c.status === 'completed').length;
const brollOk = brollClips.filter(c => c.status === 'completed').length;

return [{
  json: {
    brand: scriptData.brand || '',
    product: scriptData.product || '',
    price: scriptData.price || '',
    full_script: scriptData.script?.full_script || directorItems[0]?.json.full_script || '',
    talking_clips: talkingClips,
    broll_clips: brollClips,
    totals: {
      talking: talkingOk + '/5',
      broll: brollOk + '/5'
    },
    status: (talkingOk === 5 && brollOk === 5) ? 'ALL_COMPLETE' : 'PARTIAL',
    next_step: (talkingOk === 5 && brollOk === 5)
      ? '전체 완료. 영상 편집 진행 가능.'
      : 'URL 누락 항목 있음. missing 항목 확인 후 재실행.'
  }
}];
```

## IF 노드 조건 확인

스크린샷에서 IF 노드의 조건이 이렇게 설정되어 있다:

```
{{ $json.data.response }}  →  is not empty
```

**이 조건은 맞다.** 스크린샷의 INPUT 데이터를 보면:

- `data.response` 안에 `taskId`, `resolution`, `resultUrls` 등이 있다
- `response`가 존재하니까 → **TRUE**로 간다
- 만약 영상 생성이 안 끝났으면 `response: null` → **FALSE**로 간다

**IF 조건은 수정할 필요 없다. 출력 연결만 하면 된다.**

## 실제 데이터 구조 (스크린샷에서 확인)

B-Roll 결과 조회의 실제 응답 구조를 스크린샷에서 정확히 확인했다:

```
data
  ├── taskId: "47ff7cb24190af884c39633365b197da"
  ├── paramJson: (프롬프트, 모델 등)
  └── response
       ├── taskId: "47ff7cb24190af884c39633365b197da"
       ├── resolution: "720p"
       ├── originUrls: [null]
       ├── resultUrls
       │    └── [0]: "https://tempfile.aiquickdraw.com/v/47ff7cb...mp4"  ← 이게 영상 URL
       ├── hasAudioList: [true]
       └── seeds: [14245]
successFlag: 1
completeTime: 1772679034000
```

**핵심: 영상 URL 경로는 `$json.data.response.resultUrls[0]`이다.**

위 코드의 `extractVideoUrl` 함수에서 **경로 A**가 정확히 이 구조를 읽는다:
```javascript
if (d.data && d.data.response && d.data.response.resultUrls && d.data.response.resultUrls[0]) {
  return d.data.response.resultUrls[0];
}
```

## 만약 에러가 나면

### 에러: "Referenced node '토킹 클립 결과 조회' doesn't exist"

→ 노드 이름이 다른 것이다. n8n 캔버스에서 InfiniteTalk 결과를 조회하는 HTTP Request 노드의 **정확한 이름**을 확인해서 코드의 `$('토킹 클립 결과 조회')`를 그 이름으로 변경한다.

### 에러: "Referenced node 'B-Roll 결과 조회' doesn't exist"

→ 마찬가지. B-Roll Poll 노드의 정확한 이름을 확인해서 코드의 `$('B-Roll 결과 조회')`를 변경한다.

### talking_clips의 video_url이 전부 빈 문자열

→ `$('토킹 클립 결과 조회').all()` 또는 `$('InfiniteTalk 결과 조회').all()`이 빈 배열을 반환하고 있다. 토킹 클립 결과 조회 노드의 정확한 이름을 확인한다.
