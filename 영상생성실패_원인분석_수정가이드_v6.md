## 문제 1: TEXT_2_VIDEO 항목에 빈 image_url을 보내고 있다

### 현재 Veo 3.1 Body

```json
{
  "prompt": "{{ $json.prompt }}",
  "model": "veo3_fast",
  "generationType": "{{ $json.generation_type }}",
  "aspect_ratio": "9:16",
  "image_url": "{{ $json.image_url }}",
  "enableTranslation": false
}
```

### 문제점

B-Roll 5개 중 일부는 `generation_type: TEXT_2_VIDEO` (텍스트만으로 영상 생성)이고, 일부는 `generation_type: IMAGE_2_VIDEO` (제품 이미지 기반 영상 생성)이다.

TEXT_2_VIDEO 항목의 경우 `image_url`이 빈 문자열 `""`이다.

kie.ai API에 이렇게 전송된다:

```json
{
  "prompt": "카메라 무빙 어쩌구...",
  "model": "veo3_fast",
  "generationType": "TEXT_2_VIDEO",
  "aspect_ratio": "9:16",
  "image_url": "",
  "enableTranslation": false
}
```

API가 `code: 200`을 반환해서 **접수는 되지만**, 내부적으로 빈 `image_url` 때문에 영상 렌더링이 실패하거나 무기한 대기 상태에 빠질 수 있다. 이것이 **`response: null`이 나오는 핵심 원인**이다.

### 수정 방법

B-Roll 영상 생성 (Veo 3.1) 노드의 Body를 조건부로 바꿔야 한다. 하지만 n8n HTTP Request 노드의 JSON Body에서 조건부 필드 삭제는 expression으로는 어렵다.

**해결책: B-Roll 프롬프트 5개 생성 Code 노드에서 body를 미리 조립해서 전달한다.**

#### Step 1: B-Roll 프롬프트 5개 생성 코드 수정

기존 코드에서 return 부분을 아래처럼 수정한다. `veo_body` 필드를 추가해서, 각 항목별로 API에 보낼 body를 미리 완성한다:

```javascript
const talkingResults = $input.all();
const directorItems = $('Code in JavaScript1').all();
const scriptData = $('대본 파싱').first().json;

const clips = talkingResults.map((item, i) => {
  let videoUrl = '';
  try {
    const rd = item.json.data || item.json;
    if (rd.response && rd.response.resultUrls) {
      videoUrl = rd.response.resultUrls[0];
    } else if (rd.response && rd.response.resultUris) {
      videoUrl = rd.response.resultUris[0];
    } else if (rd.resultJson) {
      const parsed = typeof rd.resultJson === 'string' ? JSON.parse(rd.resultJson) : rd.resultJson;
      videoUrl = parsed.video?.url || parsed.url || parsed.output?.[0] || '';
    }
  } catch(e) {
    videoUrl = 'parse_error: ' + e.message;
  }
  return { segment: i + 1, video_url: videoUrl };
});

const productImageMain = scriptData.product_image_main || '';
const PRODUCT_KEYWORDS = ['제품', '클로즈업', '히어로', 'hero', 'product', 'bottle', 'KINOHI', '키노히', 'santal', '엔딩'];

const brolls = directorItems.map((item, i) => {
  const name = item.json.broll_name || '';
  const prompt = item.json.broll_prompt || '';
  const combined = (name + ' ' + prompt).toLowerCase();
  const isProductScene = PRODUCT_KEYWORDS.some(kw => combined.includes(kw.toLowerCase()));

  // ★ 핵심: TEXT_2_VIDEO와 IMAGE_2_VIDEO용 body를 각각 다르게 조립
  let veoBody;
  if (isProductScene && productImageMain) {
    // IMAGE_2_VIDEO: image_url 포함
    veoBody = {
      prompt: item.json.broll_prompt,
      model: 'veo3_fast',
      generationType: 'IMAGE_2_VIDEO',
      aspect_ratio: '9:16',
      image_url: productImageMain,
      enableTranslation: false
    };
  } else {
    // TEXT_2_VIDEO: image_url 필드 자체를 빼버린다
    veoBody = {
      prompt: item.json.broll_prompt,
      model: 'veo3_fast',
      generationType: 'TEXT_2_VIDEO',
      aspect_ratio: '9:16',
      enableTranslation: false
    };
  }

  return {
    json: {
      id: 'broll_' + (i + 1),
      name: name || ('B-Roll ' + (i + 1)),
      prompt: item.json.broll_prompt,
      generation_type: isProductScene ? 'IMAGE_2_VIDEO' : 'TEXT_2_VIDEO',
      image_url: isProductScene ? productImageMain : '',
      product_image_url: productImageMain,
      veo_body: JSON.stringify(veoBody),
      talking_clips: clips,
      brand: item.json.brand || scriptData.brand || '',
      product: item.json.product || scriptData.product || '',
      full_script: item.json.full_script || ''
    }
  };
});

return brolls;
```

#### Step 2: B-Roll 영상 생성 (Veo 3.1) Body 변경

현재 Body:
```json
={
  "prompt": "{{ $json.prompt }}",
  "model": "veo3_fast",
  "generationType": "{{ $json.generation_type }}",
  "aspect_ratio": "9:16",
  "image_url": "{{ $json.image_url }}",
  "enableTranslation": false
}
```

이것을 아래로 **전체 교체**:

```
={{ $json.veo_body }}
```

이렇게 하면:
- TEXT_2_VIDEO 항목 → `image_url` 필드가 **아예 없는** 깨끗한 body가 전송됨
- IMAGE_2_VIDEO 항목 → `image_url` 필드가 **제품 이미지 URL과 함께** 포함된 body가 전송됨

**주의**: Body Content Type은 그대로 **JSON** 으로 유지한다. `Specify Body`도 **Using JSON** 으로 유지.

하지만 이 방법에서 expression으로 JSON string을 보내는게 깨질 수 있다. 더 안전한 방법:

#### Step 2 대안: Body를 Expression이 아닌 개별 필드로 전송

Body를 JSON에서 **Using Fields** (Key-Value Pairs)로 바꾸는 것은 n8n HTTP Request에서 JSON body에 조건부 필드를 넣기 어려우므로, **가장 확실한 방법**은:

B-Roll 영상 생성 (Veo 3.1) 노드를 **2개로 분리**하는 것이다:

```
[B-Roll 프롬프트 5개 생성]
     ↓
[IF: generation_type 체크]
     ├── TEXT_2_VIDEO → [Veo 3.1 - 텍스트] (image_url 없는 body)
     └── IMAGE_2_VIDEO → [Veo 3.1 - 이미지] (image_url 있는 body)
```

하지만 이건 워크플로우 구조를 크게 바꿔야 하므로, **지금은 Step 2의 `{{ $json.veo_body }}` 방식을 먼저 시도**하고 안 되면 분리한다.

## 문제 2: 120초가 부족한 경우가 있다

### 증거

스크린샷 2에서 Run 5 of 5:
- `createTime: 1772691047000` (POST 접수 시간)
- `completeTime: null` (아직 미완성)
- `successFlag: 0`

120초 후 폴링했는데 아직 영상이 안 만들어졌다.

### 왜 이런 일이 발생하는가

- **IMAGE_2_VIDEO**는 이미지를 분석한 후 영상을 만들어야 하므로 **TEXT_2_VIDEO보다 오래 걸린다**
- kie.ai 서버의 부하에 따라 소요 시간이 달라진다 (120초에 되기도 하고, 180초 넘기기도 한다)
- 프롬프트가 길고 복잡할수록 렌더링 시간이 늘어난다

### 수정 방법: IF FALSE일 때 재폴링 (재시도)

현재 구조:
```
B-Roll 결과 조회 → IF → (FALSE) → B-Roll 순차 생성 루프 (그냥 넘어감)
```

**FALSE면 URL 없이 그냥 다음 item으로 넘어가버린다.** 이게 video_url이 비어있는 직접적 원인이다.

수정할 구조:
```
B-Roll 결과 조회 → IF → (FALSE) → [추가 대기 60초] → B-Roll 결과 조회 (재폴링)
```

#### 구체적 수정 방법

**새 노드 1개 추가: B-Roll 추가 대기 60초**

1. **+** 버튼 → 검색: **Wait** → Wait 노드 추가
2. 노드 이름: `B-Roll 추가 대기 60초`
3. 설정:
   - Resume: **After Time Interval**
   - Wait: **60**
   - Unit: **Seconds**

**연결 변경:**

| 변경 전 | 변경 후 |
|---------|---------|
| IF → (FALSE) → B-Roll 순차 생성 루프 | IF → (FALSE) → **B-Roll 추가 대기 60초** → **B-Roll 결과 조회** |

즉:
1. IF 노드의 **FALSE** 출력에서 B-Roll 순차 생성 루프로 가는 **기존 연결을 끊는다**
2. IF **FALSE** → `B-Roll 추가 대기 60초` (새 Wait 노드)에 연결
3. `B-Roll 추가 대기 60초` → `B-Roll 결과 조회`에 연결 (이미 존재하는 노드)

이렇게 하면:
- 120초 후 폴링 → 아직 안 됨 (FALSE) → 60초 더 기다림 → 다시 폴링 → 됐으면 TRUE → 루프 계속

**완성 구조:**

```
B-Roll 영상 생성 (Veo 3.1)
     ↓
B-Roll 대기 120초
     ↓
B-Roll 결과 조회 ←─────────────────────┐
     ↓                                  │
IF (response 있는가?)                    │
     ├── TRUE → B-Roll 순차 생성 루프    │
     └── FALSE → B-Roll 추가 대기 60초 ──┘
```

**주의: 무한 루프 방지**

이 구조에서는 영상이 영원히 안 되면 무한 루프에 빠질 수 있다. 하지만 kie.ai 영상 생성은 보통 최대 5분(300초) 안에 완료되므로:
- 첫 대기: 120초
- 재시도 1: +60초 = 180초
- 재시도 2: +60초 = 240초
- 재시도 3: +60초 = 300초 (5분)

5분 안에 안 되면 kie.ai 서버 문제이므로, 일단 이 구조로 가고 나중에 재시도 횟수 제한이 필요하면 추가한다.

## 문제 3: B-Roll 결과 조회의 URL expression이 재폴링 시 깨질 수 있다

### 현재 B-Roll 결과 조회 URL

```
=https://api.kie.ai/api/v1/veo/record-info?taskId={{ $json.data.taskId }}
```

### 문제점

이 expression은 **이전 노드의 `$json`** 을 참조한다.

- 첫 번째 폴링: B-Roll 대기 120초 → B-Roll 결과 조회 → `$json`은 Veo 3.1 응답 → `$json.data.taskId` ✓
- **재폴링 (문제 2의 수정 적용 시)**: B-Roll 추가 대기 60초 → B-Roll 결과 조회 → `$json`은 **이전 B-Roll 결과 조회의 응답** → `$json.data.taskId` ✓ (결과 조회 응답에도 data.taskId가 있다)

스크린샷 2의 B-Roll 결과 조회 응답을 보면:
```
data.taskId: "4fe3a2193d30e5d7693a7325efbfd278"
```

결과 조회 응답 안에도 `data.taskId`가 있으므로, **재폴링 시에도 같은 taskId로 조회된다.** 이 부분은 문제없다.

## 수정 체크리스트 (순서대로)

### 수정 A: B-Roll Body에서 빈 image_url 제거

- [ ] **B-Roll 프롬프트 5개 생성** 코드를 위의 새 코드로 **전체 교체** (`veo_body` 필드 추가)
- [ ] **B-Roll 영상 생성 (Veo 3.1)** Body를 `={{ $json.veo_body }}`로 **전체 교체**
- [ ] 테스트: B-Roll 영상 생성 노드를 Execute step으로 실행 → OUTPUT에 `code: 200` 확인

만약 `{{ $json.veo_body }}` 방식이 에러가 나면 (JSON 파싱 실패 등):

**대안**: Body를 그대로 두되, `image_url` 줄만 조건부로 변경:

```json
={
  "prompt": "{{ $json.prompt }}",
  "model": "veo3_fast",
  "generationType": "{{ $json.generation_type }}",
  "aspect_ratio": "9:16",
  {{ $json.generation_type === 'IMAGE_2_VIDEO' ? '"image_url": "' + $json.image_url + '",' : '' }}
  "enableTranslation": false
}
```

이것도 안 되면, 가장 안전한 방법:

Body를 현재 그대로 두고, TEXT_2_VIDEO 항목의 `image_url`에 빈 문자열 대신 **아무 더미 값이라도** 넣지 말고 아예 **body를 2가지 버전으로 분리**한다. 이건 워크플로우 구조를 바꿔야 하니 마지막 수단이다.

### 수정 B: IF FALSE 재폴링 루프 추가

- [ ] **B-Roll 추가 대기 60초** Wait 노드 추가 (60초)
- [ ] IF **FALSE** → B-Roll 순차 생성 루프 **기존 연결 끊기**
- [ ] IF **FALSE** → **B-Roll 추가 대기 60초** 연결
- [ ] **B-Roll 추가 대기 60초** → **B-Roll 결과 조회** 연결

### 수정 C: 최종 집계 + 업로드 코드 확인

수정 A, B를 적용하면 B-Roll URL이 정상적으로 들어올 것이다. 그러면 최종 결과 집계에서 `broll_clips[].video_url`이 채워지고, Code in JavaScript2도 에러 없이 실행된다.

## 요약: 왜 영상이 안 만들어지는가

| 원인 | 설명 | 심각도 |
|------|------|:------:|
| **빈 image_url** | TEXT_2_VIDEO에 `"image_url": ""` 보내면 kie.ai가 혼란 → 영상 렌더링 실패/무한대기 | **치명적** |
| **120초 부족** | IMAGE_2_VIDEO는 120초 안에 안 끝날 수 있음. FALSE로 빠지면 URL 없이 넘어감 | **높음** |
| **재시도 없음** | IF FALSE면 그냥 다음 item으로 넘어감 → video_url 영구 누락 | **높음** |

## 왜 1초 만에 끝나는가

**정상이다.** B-Roll 영상 생성 (Veo 3.1) POST 요청은 "이 영상 만들어줘"라고 **접수만 하는 것**이다. kie.ai 서버가 `taskId`를 즉시 반환하고, 영상 렌더링은 백그라운드에서 2~5분 동안 진행된다. 120초 대기 후 결과 조회하는 것이 바로 그 렌더링이 끝났는지 확인하는 과정이다.

## 왜 IF에서 FALSE로 가는가

120초 기다린 후 B-Roll 결과 조회를 했는데 `response: null` (아직 안 됨)이니까 IF 조건 `$json.data.response is not empty`가 거짓 → FALSE로 빠지는 것이다. 영상이 아직 렌더링 중이거나, 빈 image_url 때문에 렌더링 자체가 실패한 상태.
