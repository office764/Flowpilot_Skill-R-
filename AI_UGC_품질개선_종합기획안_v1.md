## 1. 현재 상태 분석 (AS-IS)

### 1-1. 현재 워크플로우 기술 스택

| 구성 요소 | 현재 사용 툴 | 사양 |
|-----------|-------------|------|
| TTS (음성 합성) | Typecast API (ssfm-v30) | 한국어, 단일 모델 |
| 토킹 아바타 | InfiniteTalk (kie.ai) | 720p, model: infinitalk/from-audio |
| B-Roll 영상 | Veo 3.1 (kie.ai) | 720p, 5초 단위 |
| 캐릭터 이미지 | Flux Kontext (kie.ai) | 정적 이미지 생성 |
| 배경/제품 이미지 | Flux 2 Pro (Weavy) | 제품 촬영 스타일 |
| 최종 편집 | 워크플로우 내 자동 병합 | 5개 세그먼트 연결 |
| 업로드 | Google Drive API | 자동 업로드 |

### 1-2. 현재 워크플로우 출력 품질 진단

**해상도:** 720x1280 (720p) — 인스타 릴스 기준으로도 낮은 편
**프레임레이트:** 24fps
**클립 길이:** 세그먼트당 ~5초, 총 30초 목표

**핵심 품질 문제점:**

1. **립싱크(Lip Sync) 품질 저하**
   - InfiniteTalk는 오디오 → 입 모양 매핑 수준이 기본적
   - 자연스러운 미세표정(micro-expression) 부재
   - 눈 깜빡임, 고개 움직임이 부자연스럽거나 반복적
   - 말할 때 턱/볼의 자연스러운 근육 움직임 부족

2. **아바타 리얼리즘 부족**
   - Flux Kontext 이미지 기반 → InfiniteTalk 애니메이션: 2단계 파이프라인의 한계
   - 피부 텍스처, 모공, 잔털 등 극사실주의 디테일 부족
   - 조명과 그림자가 정적(움직임에 따른 동적 조명 변화 없음)
   - 머리카락 물리학(hair physics) 부재

3. **B-Roll 품질**
   - Veo 3.1로 생성한 제품 B-Roll은 비교적 양호
   - 그러나 "사람이 제품을 사용하는" 인터랙션 장면은 생성 불가
   - 미스트 분사, 피부 도포 등 UGC 핵심 장면을 AI로 만들지 못함

4. **TTS 음성 자연스러움**
   - Typecast ssfm-v30: 한국어 지원은 되지만 감정 표현이 제한적
   - 실제 리뷰어의 톤(흥분, 감탄, 진솔한 조언)을 완벽히 재현하지 못함
   - 음성과 립싱크 간 동기화 정밀도 부족

5. **영상 구성/편집**
   - 5개 세그먼트를 단순 연결하는 구조
   - 트랜지션 효과, 텍스트 오버레이, 자막 자동 삽입 부재
   - 인트로/아웃트로 CTA(Call-to-Action) 없음

## 2. 레퍼런스 분석 (TO-BE)

### 2-1. 첨부 영상 7개 프레임별 분석

#### 영상 A: 여성 미소 씬 (720x1280, 8초, 24fps)
- **장면:** 한국 여성이 따뜻한 실내 조명 아래 자연스럽게 미소 짓는 모습
- **품질 수준:** 실제 촬영과 구분 불가능 수준
- **디테일:** 자연스러운 피부 텍스처(모공 보임), 눈가 주름, 치아의 미세한 비대칭, 머리카락의 자연스러운 웨이브와 볼륨, 배경의 자연스러운 보케(bokeh)
- **조명:** 골든아워 스타일 따뜻한 간접 조명, 피부의 서브서피스 스캐터링 효과

#### 영상 B: 침대 기지개 씬 (720x1280, 8초, 24fps)
- **장면:** 여성이 밝은 자연광 아래 침대에서 기지개를 켜는 모닝 루틴
- **품질 수준:** 실제 촬영과 구분 불가능
- **디테일:** 자연스러운 팔 동작과 근육 움직임, 이불/커튼의 사실적인 텍스처, 빛이 커튼을 통과하는 자연스러운 조명 효과
- **특이점:** 전신 동작이 포함된 장면 — 현재 우리 파이프라인에서 절대 불가능한 수준

#### 영상 C: 미스트 분사 씬 (720x1280, 8초, 24fps)
- **장면:** 여성이 팔목에 바디미스트를 뿌리는 클로즈업
- **품질 수준:** 제품 인터랙션이 극사실적
- **디테일:** 손가락의 자연스러운 관절 움직임, 팔의 잔털이 빛에 비치는 모습, 미스트 입자가 퍼지는 사실적 파티클 효과, 따뜻한 골든아워 색온도
- **핵심:** 사람+제품 인터랙션 장면 — AI UGC의 핵심 차별화 포인트

#### 영상 D: 제품 B-Roll (720x1280, 8초, 24fps)
- **장면:** 대리석 위에 놓인 향수병에서 미스트가 피어오르는 분위기 있는 제품샷
- **품질 수준:** 프로급 제품 사진/영상 수준
- **디테일:** 유리병의 사실적인 빛 굴절과 반사, 미스트의 자연스러운 확산, 식물 소품의 디테일, 대리석 표면의 텍스처

#### 영상 E: 피부 고민 씬 (720x1280, 8초, 24fps)
- **장면:** 건조하고 각질이 일어난 팔 피부를 클로즈업으로 보여주는 "Before" 장면
- **품질 수준:** 의료/다큐멘터리 수준의 극사실적 클로즈업
- **디테일:** 각질, 피부 결, 모공, 잔털이 개별적으로 식별 가능, 자연스러운 피부색의 불균일함

#### 영상 F: 동일 영상의 다른 세그먼트 (위 영상 A와 동일 파일)

#### 영상 G: VIDI 클렌징 디바이스 영상 (1080x1896, 27초, 30fps)
- **장면:** 한국 여성이 욕실에서 클렌징 패드로 얼굴을 닦는 모습
- **명시적 표기:** "VIDI를 사용해 100% AI로 생성된 영상입니다"
- **품질 수준:** 실제 촬영과 완전히 구분 불가능 — 현재 AI UGC 업계 최고 수준
- **디테일:**
  - 해상도 1080x1896 (풀HD+ 세로형)
  - 30fps 부드러운 모션
  - 자연스러운 손-얼굴 인터랙션 (패드로 볼을 쓸어내리는 동작)
  - 피부에 닿는 패드의 압력감이 사실적
  - 미세한 눈 움직임과 자연스러운 표정 변화
  - 욕실 타일 배경의 사실적인 텍스처
  - 목욕가운의 체크 패턴이 움직임에 따라 자연스럽게 변형
  - 한글 자막 "분명 꼼꼼하게 씻었는데" 오버레이
- **핵심 시사점:** VIDI는 "사람이 제품을 직접 사용하는" 풀 인터랙션 영상을 AI로 생성할 수 있음

### 2-2. VIDI 서비스 분석

**VIDI (비디)** — "촬영없이 만드는 AI 광고 숏폼"

- K-뷰티 브랜드 타겟의 AI UGC 비디오 생성 솔루션
- 인플루언서 시딩 없이 고성능 숏폼 광고 영상 제작
- 제품 이미지 1장으로 완성형 UGC 영상 생성
- 1080p+ 해상도, 30fps, 최대 30초+
- 사람+제품 인터랙션이 포함된 풀바디 아바타 영상 생성 가능
- 한글 자막 자동 삽입
- 뷰티/코스메틱 전문 특화 모델로 학습되어 피부, 메이크업, 스킨케어 장면에 특화

## 3. 갭 분석 (AS-IS vs TO-BE)

### 3-1. 품질 비교표

| 항목 | 현재 (AS-IS) | 목표 (TO-BE / VIDI 수준) | 갭 수준 |
|------|-------------|------------------------|--------|
| **해상도** | 720x1280 (720p) | 1080x1920+ (1080p+) | 심각 |
| **프레임레이트** | 24fps | 30fps+ | 중간 |
| **립싱크 품질** | 기본 (입만 움직임) | 극사실적 (미세표정+몸짓) | 심각 |
| **아바타 리얼리즘** | 중하 (AI티 남) | 실사 구분불가 | 심각 |
| **제품 인터랙션** | 불가능 | 자연스러운 사용 장면 | 심각 |
| **전신 동작** | 불가능 (얼굴만) | 상반신 이상 자연스러운 동작 | 심각 |
| **TTS 자연스러움** | 중간 (기계적) | 높음 (감정+억양 자연스러움) | 중간 |
| **B-Roll 품질** | 양호 (Veo 3.1) | 시네마틱 수준 | 낮음 |
| **자막/오버레이** | 없음 | 한글 자막 자동 삽입 | 중간 |
| **트랜지션/편집** | 단순 연결 | 부드러운 컷/트랜지션 | 중간 |
| **영상 길이** | 5초 x 5 = 25초 | 연속 30초+ | 낮음 |

### 3-2. 핵심 갭 3가지

**GAP #1 — 아바타 리얼리즘 & 립싱크 (최우선)**
현재 InfiniteTalk(kie.ai)의 한계가 전체 품질의 병목. 얼굴만 움직이고, 미세표정이 없으며, 움직임이 부자연스러움. VIDI 수준으로 가려면 아바타 모델 자체를 교체해야 함.

**GAP #2 — 제품 인터랙션 장면 생성 불가 (핵심 차별점)**
현재 워크플로우에는 "사람이 제품을 사용하는" 장면을 생성할 수단이 전혀 없음. VIDI 레퍼런스에서 가장 인상적인 부분이 바로 이 제품-사람 인터랙션. 이것은 단순 아바타 교체로 해결되지 않고, 별도의 이미지→비디오 파이프라인이 필요.

**GAP #3 — 해상도 & 전반적 프로덕션 퀄리티**
720p → 1080p+ 업그레이드, 자막 자동 삽입, 트랜지션 효과, 색보정 등 포스트프로덕션 요소가 모두 부재.

### 4-3. 제품 인터랙션 영상 생성 (신규 파이프라인 필요)

현재 워크플로우에 가장 크게 빠져있는 부분. "사람이 제품을 사용하는" 장면을 만들기 위한 방법론:

**방법 A: Kling 3.0 Image-to-Video + 합성 키프레임**
- Weavy에서 캐릭터+제품 합성 이미지 생성 (SD3 Remove BG → Compositor)
- 합성 키프레임을 Kling 3.0 Image-to-Video의 First Frame으로 입력
- 동작 프롬프트로 "spraying mist on wrist" 등 지시
- 장점: 현재 Weavy 파이프라인과 호환
- 단점: 복잡한 인터랙션(제품을 집어서 뿌리는 등)은 부자연스러울 수 있음

**방법 B: Higgsfield AI 멀티모델 플랫폼**
- Higgsfield는 Kling 3.0, Sora 2, WAN 2.5를 통합한 플랫폼
- AI Influencer Generator로 일관된 캐릭터 생성
- Image-to-Video로 제품 인터랙션 장면 생성
- 장점: 올인원 플랫폼, 캐릭터 일관성 유지
- 단점: 세밀한 API 제어 가능 여부 확인 필요

**방법 C: VIDI 직접 사용 (가장 확실)**
- VIDI가 뷰티/코스메틱 UGC에 가장 특화된 솔루션
- 제품 이미지 1장으로 인터랙션 장면 자동 생성
- 단점: 자체 API/자동화 연동 가능 여부 미확인 (현재 공개 API 없을 가능성 높음)
- 액션: VIDI 팀에 직접 컨택하여 API/화이트라벨 파트너십 문의 필요

### 4-4. 영상 후처리 & 편집 자동화

| 기능 | 추천 툴 | 방법 |
|------|---------|------|

## 5. 개선 로드맵 (3단계)

**목표:** 현재 워크플로우 구조 유지하면서 최대한 품질 끌어올리기

   - n8n HTTP Request 노드의 엔드포인트/바디만 변경

   - Form Trigger에서 해상도 선택 옵션 추가

4. **자막 자동 삽입 추가**
   - 최종 결과 집계 후 FFmpeg로 TTS 텍스트 기반 자막 오버레이
   - 스타일: VIDI 레퍼런스처럼 하단 한글 자막

### Phase 2: 구조 개선 (2~4주) — "Game Changer"

**목표:** 제품 인터랙션 장면 추가, 영상 구성 다변화

6. **영상 편집 자동화 강화**

   - 타입별 다른 생성 파이프라인으로 자동 라우팅

### Phase 3: 프리미엄 진입 (4~8주) — "VIDI Killer"

**목표:** VIDI 수준 혹은 그 이상의 품질로 시장 진입

   - VIDI에 API/화이트라벨 제안 → 우리 n8n 워크플로우에 통합

## 7. TTS 심층 분석 및 개선안

### 7-1. 현재 Typecast 문제점

1. **감정 레인지 제한:** ssfm-v30 모델은 "평이한 리뷰 톤"만 가능. 흥분("이거 대박이에요!"), 속삭임("은은한 편백 향이..."), 진솔한 조언("제가 직접 써봤는데요") 등의 감정 변화를 표현 못함
2. **억양 자연스러움:** 한국어 특유의 리듬감(높낮이 변화)이 부족. 로봇틱한 느낌
3. **호흡/포즈:** 자연스러운 쉼(pause), 숨소리(breath) 없이 단조로운 진행

### 7-2. ElevenLabs 전환 시 개선되는 점

1. **감정 톤 제어:** API 파라미터에서 stability, similarity_boost, style 조절 가능
2. **한국어 특화 음성:** 다양한 한국어 네이티브 음성 모델 제공 (여성/남성, 연령대별)
3. **보이스 클로닝:** 실제 리뷰어 목소리 샘플 3분만 있으면 그 목소리로 TTS 생성 가능 → 브랜드별 전속 AI 리뷰어 목소리 확보
4. **SSML 지원:** 강조, 속도 변경, 포즈 삽입 등 세밀한 제어
5. **Creatify Aurora 네이티브 통합:** ElevenLabs 음성 → Aurora 아바타로 바로 연결 가능 (별도 오디오 파일 전달 불필요)

### 7-3. TTS 전환 시 n8n 워크플로우 변경점

**현재 Typecast 호출:**
```
POST https://api.typecast.ai/v1/text-to-speech
Body: { "text": "...", "model": "ssfm-v30", ... }
```

**변경 후 ElevenLabs 호출:**
```
POST https://api.elevenlabs.io/v1/text-to-speech/{voice_id}
Headers: { "xi-api-key": "{API_KEY}" }
Body: {
  "text": "...",
  "model_id": "eleven_multilingual_v2",
  "voice_settings": {
    "stability": 0.5,
    "similarity_boost": 0.75,
    "style": 0.4,
    "use_speaker_boost": true
  }
}
```

변경 노드: `세그먼트별 TTS 생성 (Typecast)` → `세그먼트별 TTS 생성 (ElevenLabs)`로 URL, Header, Body만 변경. 나머지 워크플로우 구조는 동일 (오디오 URL을 받아서 다음 노드로 전달하는 패턴).

## 10. 결론 및 다음 단계

현재 우리 워크플로우의 AI UGC 품질은 VIDI 레퍼런스 대비 **체감 30% 수준**이다. 가장 큰 원인은 InfiniteTalk의 립싱크/아바타 품질 한계와 제품 인터랙션 장면 부재다.

**즉각 실행할 것:** fal.ai API를 통한 Creatify Aurora + ElevenLabs TTS 교체만으로 품질 3~4배 향상이 가능하며, 이는 n8n 워크플로우의 HTTP Request 노드 URL/Body만 변경하면 되는 수준이다. 워크플로우 구조 자체를 바꿀 필요 없이 "드롭인 교체"가 가능하다.

**장기적으로:** VIDI 파트너십 또는 자체 풀 파이프라인 구축으로 VIDI를 넘어서는 것이 목표.

이 기획안의 내용은 워크플로우를 건드리지 않고, 대표님의 승인 후에만 순차적으로 적용할 예정이다.
