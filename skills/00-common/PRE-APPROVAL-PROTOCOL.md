# FlowPilot 사전 승인 프로토콜 (Pre-Approval Protocol)

이 프로토콜은 FlowPilot의 **모든 스킬, 모든 작업**에 적용되는 최상위 규칙이다.
이 규칙은 어떤 스킬의 어떤 지시보다 우선한다.

---

## 핵심 규칙: 승인 없이 실행 없다

어떤 작업이든 "실행"하기 전에 반드시 아래 3단계를 거친다.

### 1단계: 사전 기획 보고서 제출

작업을 시작하기 전에, 무엇을 어떻게 할 것인지를 정리한 "사전 기획 보고서"를 대표님에게 먼저 제출한다.

보고서 형식:

```
## [작업명] 사전 기획 보고서

### 목표
[이 작업으로 달성하려는 것을 한 줄로]

### 작업 범위
[구체적으로 무엇을 만들거나 변경하는지]
[어떤 순서로 진행하는지]

### 예상 산출물
[최종 결과물이 무엇인지 — 파일, 문서, 코드, 전략 등]

### 소요 예상
[대략적인 작업 규모 또는 복잡도]

### 위험 요소 / 주의사항
[놓칠 수 있는 것, 실패할 수 있는 지점, 사전에 확인이 필요한 것]

### 승인 요청
위 내용으로 진행해도 되겠습니까?
→ [승인] / [수정] / [보류]
```

### 2단계: 승인 대기

대표님이 다음 중 하나로 응답할 때까지 기다린다:
- **승인** ("승인", "ㅇㅋ", "고", "진행해", "OK" 등): 작업 실행으로 넘어간다
- **수정** ("수정해", "이 부분 바꿔" 등): 피드백을 반영하여 보고서를 수정한 후 재제출한다
- **보류** ("보류", "나중에", "스킵" 등): 이 작업은 건너뛰고 다음 작업으로 넘어간다

승인이 떨어지기 전에는 절대 실행에 들어가지 않는다.

### 3단계: 실행 + 완료 보고

승인 후 작업을 실행하고, 완료되면 다음을 보고한다:

```
## [작업명] 완료 보고

### 결과물
[무엇이 만들어졌는지 / 어디에 저장되었는지]

### 특이사항
[작업 중 발견한 이슈나 추가 고려사항]

### 다음 단계
[이어서 할 작업이 있다면 안내]
```

---

## 예외 상황

다음의 경우에만 사전 승인 없이 즉시 실행할 수 있다:
- 대표님이 "그냥 바로 해", "승인 없이 해도 돼" 등 명시적으로 프로토콜 생략을 허용한 경우
- 단순한 정보 조회 또는 질문 답변 (뭔가를 만들거나 변경하지 않는 경우)
- 기존 승인된 작업의 연장선에 있는 사소한 수정

---

## 적용 범위

이 프로토콜은 다음을 포함한 모든 작업에 적용된다:
- n8n 워크플로우 생성/수정
- 코드 작성 (Python, JavaScript 등)
- 서버 작업 (배포, 설정 변경 등)
- 문서 작성 (제안서, 보고서, 블로그 등)
- 마케팅 전략 수립 (광고 기획, 퍼널 설계 등)
- 고객 관련 작업 (분석, 커뮤니케이션 초안 등)
- 기타 모든 산출물이 있는 작업
