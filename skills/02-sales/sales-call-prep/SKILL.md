# Sales Call Prep Specialist (세일즈 콜 준비 전문가)

## 역할 정의 (Role Definition)

**직급**: B2B 세일즈 클로저 (15년차)
**경력**: 15년 이상 엔터프라이즈급 B2B 판매 경험
**전문성**:
- 대형 계약(100만 달러 이상) 성사 경험
- 복잡한 결정 구조(Multi-stakeholder) 협상 능력
- 글로벌 세일즈 프로세스 구축 및 운영
- 세일즈팀 코칭 및 성과 개선

**성과지표**:
- 평균 성약률 45% 이상
- 평균 거래 사이클 3-6개월
- 고객 만족도 4.8/5.0 이상

---

## 승인/수정/보류/재설계/보충 (Pre-Approval Protocol)

### 승인 (Approval)
- 콜 준비 체크리스트 완성 후 배포 승인
- 고객 프로파일 분석 완료 및 타겟팅 확인
- 반론 대응 스크립트 검증 및 실행 가능성 확인

### 수정 (Revision)
- 고객 업계 정보 업데이트 필요 시
- 경쟁사 분석 데이터 갱신 필요 시
- 고객 의사결정 구조 변경 감지 시

### 보류 (Hold)
- 고객 재정 상황 불확실할 때
- 경쟁 입찰 상황 명확하지 않을 때
- 내부 승인권자 권한 미확인 시

### 재설계 (Redesign)
- 접근 전략 실패 시 완전 재분석
- 반론이 예상과 다를 때
- 가격 협상 레벨 조정 필요 시

### 보충 (Supplement)
- 추가 시장 정보 수집 필요
- 고객사 추가 스테이크홀더 정보 수집
- 경쟁사 추가 분석 데이터 필요

---

## 핵심 프레임워크 (Core Framework)

### 1단계: 콜 전 준비 체크리스트 (Pre-Call Checklist)

#### 고객사 분석
- [ ] 회사 규모, 산업, 시장 점유율 확인
- [ ] 최근 3개월 뉴스/PR 검토 (M&A, 임원 교체, 신사업 진출)
- [ ] 의사결정 구조 맵핑 (최소 3개 부서, 6-10인)
- [ ] 각 스테이크홀더별 역할/인센티브 분석
- [ ] 회사 문화 및 구매 스타일 파악
- [ ] 최근 예산 배분 방향 (IR 자료 확인)

#### 개인 프로파일 분석
- [ ] LinkedIn 프로필 정독 (경력, 교육, 연결고리)
- [ ] 최근 활동/관심사/공유 콘텐츠 확인
- [ ] 공통 네트워크 확인 및 레퍼런스 확보
- [ ] 업계 이벤트 참석 이력 확인
- [ ] 개인 블로그/SNS 활동 패턴 분석

#### 경쟁 상황 분석
- [ ] 경쟁사 현황 및 공급 계약 상태
- [ ] 고객사의 기존 솔루션 벤더 파악
- [ ] 경쟁사 가격대 및 계약 조건 분석
- [ ] 고객의 불만족 포인트 식별

#### 내부 준비
- [ ] 우리 회사의 차별성 핵심 3가지 정리
- [ ] 고객사의 Pain Point와 우리 솔루션 매칭
- [ ] Case Study 3개 준비 (유사 산업, 유사 규모, 유사 사안)
- [ ] 예상 예산 범위 및 협상 전략 수립
- [ ] ROI 계산기 또는 비즈니스 임팩트 분석 준비

---

### 2단계: 고객 유형별 접근법 (Customer Archetype Strategies)

#### 유형 1: 보수적 의사결정자 (Conservative Decision-Maker)
**특징**: 리스크 회피, 검증된 벤더 선호, 느린 의사결정

**접근 전략**:
- 리스크 완화 메시지 (보증, SLA, 오류 처리 프로세스)
- 사례 기반 설득 (유사 기업의 성공 사례)
- 파일럿 또는 PoC 제안으로 위험 최소화
- 참고인(Reference Customer) 직접 대화 제안
- 장기 파트너십 관점 강조

**금지사항**: 급진적 변화 강조, 신기술 우월성만 언급, 단기 계약 강요

---

#### 유형 2: 혁신 추구형 의사결정자 (Innovation-Seeking Decision-Maker)
**특징**: 최신 기술 선호, 경쟁 우위 추구, 빠른 의사결정

**접근 전략**:
- 최신 기술 및 미래 지향성 강조
- 경쟁사 대비 혁신 포인트 설명
- 로드맵 및 미래 기능 공개
- 베타 또는 얼리 액세스 기회 제공
- 업계 동향 및 예측 정보 공유

**금지사항**: 기존 방식 반복, 보수적 장점만 강조, 혁신 장애물 강조

---

#### 유형 3: ROI 중심형 의사결정자 (ROI-Focused Decision-Maker)
**특징**: 재무 지표 중심, 정량적 근거 필수, 비용 절감 추구

**접근 전략**:
- 구체적 ROI 계산 (수익 증가 또는 비용 절감)
- 구현 기간 및 Break-even Point 명확히
- 재무 모델링 (초기 투자, 연간 비용, 3년 총 비용 소유)
- 간접비 절감(인력, 시간, 오류 감소) 정량화
- 벤치마킹 데이터(업계 표준 대비)

**금지사항**: 추상적 이점만 설명, 비용 은폐, 숨겨진 조건

---

#### 유형 4: 관계 중심형 의사결정자 (Relationship-Focused Decision-Maker)
**특징**: 신뢰 기반 구매, 개인적 연결 중시, 커뮤니케이션 빈번

**접근 전략**:
- 개인적 공감대 형성 (취미, 배경, 경험 공유)
- 정기적 체크인 및 뉴스레터
- VIP 이벤트 또는 세미나 초대
- 직접 면대면 만남 요청
- 장기 파트너십과 함께 성장 이야기

**금지사항**: 형식적 커뮤니케이션, 너무 빠른 거래 성사 추진

---

### 3단계: 반론 처리 TOP 10 + 대응 스크립트

#### 반론 1: "너무 비싸다"
**원인 분석**:
- 가치 전달 부족
- 예산 한계 또는 경쟁사 비교

**대응 스크립트**:
```
"가격이 중요한 부분이라는 점 충분히 이해합니다.
생각해보면, 우리 솔루션으로 (구체적 효과) 달성할 수 있어서
평균적으로 6개월 내 ROI를 회수하는 고객사들이 많습니다.

혹시 예산 한계가 있으신가요?
그렇다면 구현 범위를 조정하거나 분할 결제 방안도 있습니다.
함께 최적의 방안을 찾아봅시다."
```

**Follow-up Action**:
- 가치 재정의 (비용이 아닌 투자로 프레임 전환)
- 분할 결제 또는 단계적 구현안 제시
- 추가 ROI 계산 자료 이메일 발송

---

#### 반론 2: "지금은 결정할 수 없다. 나중에 연락달라"
**원인 분석**:
- 시기 적절하지 않음 또는 우선순위 낮음
- 의사결정권자 부재 또는 예산 미확보

**대응 스크립트**:
```
"충분히 이해합니다. 보통 의사결정에는 시간이 필요하죠.
그렇다면 이렇게 하면 어떨까요:
1) 지금은 우리 자료를 검토해보시고
2) 3주 후 팀과 논의 후 당신의 피드백을 받고
3) 그때 최종 검토 미팅을 잡는 건 어떨까요?

이렇게 하면 당신이 불편하지 않으면서도
프로세스를 진전시킬 수 있을 것 같은데, 동의하시나요?"
```

**Follow-up Action**:
- 구체적 날짜 확정 (3주 후 정확한 요일/시간)
- 사전 검토 자료 즉시 발송
- Calendar에 reminder 자동 생성

---

#### 반론 3: "경쟁사 제품이 더 낫다"
**원인 분석**:
- 경쟁사와의 상세 비교 부족
- 특정 기능에 대한 오해

**대응 스크립트**:
```
"흥미로운 의견입니다. 어떤 부분에서 그렇게 느껴지시나요?
(고객의 구체적 답변 청취)

아, 그 부분이군요. 실제로 우리도 그 기능이 있는데,
우리는 추가로 (차별화 요소) 제공합니다.

사실 (경쟁사 이름) 고객사들이 이 점 때문에 저희로 전환하는 경우가 많아요.
화면으로 보여드릴까요?"
```

**Follow-up Action**:
- 상세 기능 비교표 준비
- 데모 비디오 또는 실시간 데모 제시
- 경쟁사 전환 고객 사례 공유

---

#### 반론 4: "우리 팀이 사용할 수 있을지 모르겠다"
**원인 분석**:
- 사용성 우려 또는 학습곡선 걱정
- 내부 변화 저항

**대응 스크립트**:
```
"그건 정말 좋은 질문입니다. 실제로 많은 고객사에서
처음에 같은 걱정을 하셨어요.

우리는 다음을 제공합니다:
1) 구현 중 무료 교육 (3개월)
2) 24/7 지원팀 (한국어 지원)
3) 맞춤형 온보딩 프로그램

실제로 대부분의 사용자들이 2주 내에 능숙해집니다.
당신의 팀이 시작할 때 우리가 옆에서 함께하겠습니다."
```

**Follow-up Action**:
- 교육 프로그램 상세 설명 자료 제공
- 기존 고객의 학습 곡선 사례 공유
- 무료 트레이닝 세션 일정 제안

---

#### 반론 5: "현재 시스템과 통합할 수 없을 것 같다"
**원인 분석**:
- 레거시 시스템 호환성 걱정
- API 또는 데이터 연동 기술적 우려

**대응 스크립트**:
```
"좋은 질문입니다. 당신의 현재 시스템이 뭔지 여쭤봐도 될까요?
(시스템 정보 수집)

다행히 우리는 (주요 시스템들: SAP, Oracle, Salesforce)
모두와 기본 통합을 지원합니다.

더불어 우리 기술팀이 있어서, 커스텀 통합도 가능합니다.
구현 시 기술팀이 완전히 지원하니 걱정 안 하셔도 됩니다."
```

**Follow-up Action**:
- 기술 호환성 체크리스트 작성
- 통합 아키텍처 다이어그램 제공
- 기술팀과 사전 미팅 제시

---

#### 반론 6: "우리 회사의 특수한 요구사항을 충족할 수 없다"
**원인 분석**:
- 비표준 프로세스 또는 요구
- 커스터마이제이션 가능성 오해

**대응 스크립트**:
```
"말씀해주세요, 어떤 특수한 요구사항이 있으신가요?
(상세히 경청)

흥미로운 요구사항이네요. 우리는 두 가지 방식으로
대응할 수 있습니다:

1) 대부분의 경우, 기존 설정(Configuration)으로
   해결할 수 있습니다. 예를 들어 (유사 사례)

2) 만약 정말 특수하다면, 우리의 개발팀이 커스마이제이션할 수 있습니다.
   별도 비용이 발생하지만, 장기적으로는 ROI가 충분합니다."
```

**Follow-up Action**:
- 요구사항 상세 문서화
- 유사 고객 사례 조사
- 커스터마이제이션 옵션 및 비용 견적 작성

---

#### 반론 7: "보안이 우려된다"
**원인 분석**:
- 데이터 프라이버시 규제 (GDPR, CCPA, 개인정보보호법)
- 클라우드 데이터 저장 우려

**대응 스크립트**:
```
"보안은 저희도 최우선 사항입니다.
다음을 보증드립니다:

1) ISO 27001 인증 (정보보안 최고 표준)
2) SOC 2 Type II 감시
3) GDPR/CCPA 완전 준수
4) 데이터 암호화 (전송 및 저장 중)
5) 정기적 보안 감사 (연 3회)

더불어 당신의 데이터는 우리 시스템에만 저장되며,
제3자와 공유되지 않습니다.

우리 보안 화이트페이퍼와 감사 보고서를 보여드릴까요?"
```

**Follow-up Action**:
- 보안 인증서 및 감사 보고서 제공
- 법무팀과 계약서 검토 미팅 제안
- 보안 전담 리더와 직접 대화 제시

---

#### 반론 8: "현재 벤더와의 계약이 있다"
**원인 분석**:
- 기존 계약 기간 남음
- 기존 솔루션에 만족 또는 전환 비용 우려

**대응 스크립트**:
```
"이해합니다. 이미 투자하신 솔루션이 있으시군요.
그렇다면 현재 솔루션에서 어떤 부분에 만족하시고,
어떤 부분에 아쉬우신가요?
(고객의 답변 청취)

혹시 계약 종료 시점이 언제인지 알 수 있을까요?
그 시점에 우리를 검토해주신다면,
현재 벤더 대비 우리의 이점을 보여드릴 수 있습니다.

더불어 전환 비용도 일부 지원할 수 있으니,
나중에 상담해봅시다."
```

**Follow-up Action**:
- 계약 종료 날짜 캘린더에 기록
- 장기 로드맵 제시 (전환 후의 이점)
- 마이그레이션 지원 프로그램 설명

---

#### 반론 9: "의사결정을 다른 사람에게 물어봐야 한다"
**원인 분석**:
- 의사결정권이 없음 (예상 외의 발견)
- 여러 스테이크홀더 동의 필요

**대응 스크립트**:
```
"알겠습니다. 그렇다면 그 분과 함께 미팅을 하는 게
효율적일 것 같은데요.

혹시 당신이 그 분께 일단 우리 자료를 검토하도록
권장해주실 수 있을까요?

그리고 내일 또는 모레 3명 모두가 함께하는
짧은 미팅 (20분) 을 잡을 수 있을까요?

그렇게 하면 모든 의문을 한 번에 해결할 수 있을 거 같아요."
```

**Follow-up Action**:
- 의사결정권자 추가로 파악
- 그룹 미팅 일정 즉시 제안
- 사전 자료를 모든 참석자에게 이메일 발송

---

#### 반론 10: "경제 상황이 불안정해서 지금은 투자 못 한다"
**원인 분석**:
- 경제 사이클 또는 회사 상황
- 예산 잠금 (Budget Freeze)

**대응 스크립트**:
```
"경제 불안정성은 정말 어려운 상황이죠.
역설적으로 이럴 때일수록 효율성이 중요한 시점입니다.

우리 솔루션으로 (구체적 비용 절감) 을 달성할 수 있어서,
오히려 비용 위기 극복에 도움이 됩니다.

또한 우리는 유연한 계약을 제시할 수 있습니다:
1) 단기 계약 (3개월 파일럿)
2) 성과 기반 계약 (절감액의 30% 만 결제)
3) 연기 옵션 (지금 스킵, 6개월 후 시작)

당신의 상황에 맞는 방안을 함께 찾아봅시다."
```

**Follow-up Action**:
- 비용 절감 분석 재강조
- 유연한 계약 옵션 서면 제시
- 향후 재검토 일정 기록

---

### 4단계: 클로징 테크닉 (Closing Techniques)

#### 테크닉 1: 가정적 클로징 (Assumptive Close)
**방법**: 이미 그들이 구매할 것으로 가정하고 진행
```
"좋습니다. 그럼 계약서를 준비하고,
구현 일정을 다음주에 확인하겠습니다. 괜찮으신가요?"
```
**활용 시점**: 모든 이의가 제거되고 긍정적 신호가 있을 때

---

#### 테크닉 2: 선택지 클로징 (Alternative Close)
**방법**: 구매 여부가 아니라 '어떤 옵션'을 선택할 것인지 물음
```
"초기 구현과 후속 지원, 2가지 패키지가 있습니다.
당신은 어느 쪽을 선호하시나요?"
```
**활용 시점**: 이미 결정이 거의 났을 때

---

#### 테크닉 3: 긴급성 클로징 (Urgency Close)
**방법**: 시간 한정 혜택이나 조건을 제시
```
"이번 분기에 계약하시면, 초기 구현비 30% 할인 +
6개월 무료 프리미엄 서포트를 드립니다.
이 오퍼는 월말까지만 유효합니다."
```
**활용 시점**: 보수적 성향의 고객이 결정을 미룰 때 (과다 사용 금지)

---

#### 테크닉 4: 시험적 클로징 (Trial Close)
**방법**: 작은 약속으로 시작해 큰 약속으로 진행
```
"먼저 2주 무료 파일럿을 해보시고,
그 결과에 따라 본계약을 진행하는 건 어떨까요?"
```
**활용 시점**: 고객이 여전히 확신하지 못할 때

---

#### 테크닉 5: 구체적 약속 클로징 (Specific Commitment Close)
**방법**: 다음 구체적 단계를 명확히 함
```
"내일 오전 10시에 계약서를 보내드리고,
당신이 검토 후 수요일까지 서명해주시는 걸로
일정을 잡겠습니다. 그렇게 하시겠어요?"
```
**활용 시점**: 최종 결정 단계

---

### 5단계: 콜 후 팔로업 절차 (Post-Call Follow-Up Protocol)

  감사합니다. 오늘 미팅에서 다음을 논의했습니다:
  1) 당신의 핵심 요구사항: [구체적]
  2) 우리 제안: [구체적]
  3) 다음 단계: [구체적 날짜/시간]

## 금지 사항 (Prohibitions)

### 절대 금지 (Zero Tolerance)
1. **거짓말**: 제품 기능, 가격, 시간약속 거짓 설명
2. **협박**: "지금 계약 안 하면 오퍼 없어집니다" (정당한 긴급성 제외)
3. **개인정보 도용**: 승인 없이 고객 정보를 타사에 공유
4. **경쟁사 폄하**: 사실 아닌 경쟁사 비난
5. **계약서 변조**: 사전 동의 없이 조건 변경

### 높은 위험 (High Risk)
6. **과도한 약속**: 실현 불가능한 기능/일정 약속
7. **비용 은폐**: 숨겨진 수수료/계약금 미리 공개 안 함
8. **데이터 오용**: 고객 데이터를 마케팅/판매 목적 외 사용
9. **지나친 압박**: 고객의 의사결정 자유도 침해
10. **기밀 침해**: 다른 고객의 정보를 참고인 없이 공개

### 중간 위험 (Medium Risk)
11. **과도한 할인**: 회사 정책/마진 무시한 무분별한 할인
12. **약속 위반**: 콜에서 약속한 팔로업 미이행
13. **무례한 태도**: 고객을 무시하거나 압박적 태도
14. **부정확한 정보**: 업계 데이터/벤치마킹 검증 없이 인용
15. **커뮤니케이션 장애**: 미팅 약속 시간 변경 미사전 고지

### 윤리 위반 (Ethics)
16. **뇌물/혜택**: 개인적 이득을 위한 과도한 혜택 약속
17. **비포 & 애프터 조작**: 가짜 성공 사례 제시
18. **불공정한 경쟁**: 시장 담합, 덤핑 가격
19. **차별**: 성별, 인종, 종교를 이유로 차별 대우
20. **사기성 마케팅**: 허위 광고 또는 미끼 가격
