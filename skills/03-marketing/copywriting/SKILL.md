# Copywriting — FlowPilot 전용

## 최우선 원칙: AI 티 절대 배제

이 원칙은 이 스킬의 다른 모든 원칙보다 위에 있다.

FlowPilot의 모든 마케팅 텍스트는 **100% 사람이 쓴 것처럼** 읽혀야 한다.
20년차 마케팅 카피라이터가 커피 한 잔 마시면서 자기 경험을 바탕으로 쓴 글처럼.
AI가 생성한 냄새가 나는 순간, 신뢰도는 바닥으로 떨어진다.

### AI 티가 나는 글의 특징 (절대 하지 말 것)

**문체/어투:**
- "~에 대해 알아보겠습니다" "~에 대해 살펴보겠습니다" → 전형적 AI 도입부
- "다양한", "효과적인", "최적화된", "혁신적인" → AI가 좋아하는 형용사 남발
- "~할 수 있습니다", "~하는 것이 중요합니다" → 존댓말 나열체
- "결론적으로", "종합적으로", "궁극적으로" → AI 마무리 패턴
- "~라고 할 수 있겠습니다" → 어디서 많이 본 애매한 마무리
- 모든 문장이 비슷한 길이와 리듬 → 단조로운 기계적 느낌
- 과도한 불릿포인트와 번호 매기기 → 정리는 됐는데 생동감 없음

**구조:**
- 서론-본론-결론이 너무 깔끔하게 떨어짐 → 사람은 이렇게 안 씀
- 모든 단락이 3~4문장으로 균일 → 기계적
- 접속사가 너무 매끄러움 → "또한", "더불어", "아울러" 남발

**내용:**
- 구체적 경험이나 사례 없이 일반론만 나열
- "~하면 좋습니다" "~를 추천합니다" 식의 무난한 조언
- 감정이 없음. 짜증도 없고, 흥분도 없고, 아쉬움도 없음

### 자연스러운 한국어 글쓰기 규칙

**문장 길이를 섞어라:**
짧은 문장. 그리고 갑자기 길어지는 문장이 나와도 괜찮다, 왜냐하면 사람은 원래 그렇게 말하니까. 다시 짧게.

**구어체를 적절히 섞어라:**
"솔직히 말해서", "근데 진짜", "이게 왜 중요하냐면", "아니 이걸 왜 아직도"
→ 글에 숨결이 들어간다

**불완전한 문장도 OK:**
"결과? 놀라웠다."
"그래서 뭘 했냐고요."
→ 사람은 항상 완벽한 문장으로 말하지 않는다

**감정을 넣어라:**
짜증, 놀라움, 안타까움, 기대감, 유머.
"이걸 아직도 수작업으로 하고 있다니, 솔직히 좀 안타깝다."
→ 감정이 있어야 사람이 쓴 글이다

**"나"의 경험을 넣어라:**
"저희가 실제로 물류회사 A사와 작업했을 때..."
"처음엔 저도 반신반의했는데..."
→ 경험담이 있으면 AI 티가 사라진다

**한국어 자연 문법:**
- 조사 선택: "을/를" 생략이 자연스러운 곳에선 생략한다 ("커피 마시러" vs "커피를 마시러")
- 어미 변화: "-거든요", "-잖아요", "-더라고요" 등 구어체 어미 활용
- 주어 생략: 한국어는 주어를 자주 생략한다. "저는 생각합니다"보다 "생각해 보면"이 자연스럽다

## 글쓰기 구조 공식 (상황별 선택)

### 공식 1: PAS (Problem → Agitate → Solution)
가장 범용적인 마케팅 카피 구조. 블로그, 랜딩페이지, 이메일 전부 가능.

**P (Problem) — 문제 제기:**
독자가 "아, 이거 내 얘기인데?" 하고 공감하게 만든다.
```
매일 아침 출근하면 가장 먼저 하는 게 뭔가요?
아마 이메일 열고, 어제 들어온 주문 확인하고, 엑셀 열어서 하나하나 옮겨 적는 거 아닌가요?
```

**A (Agitate) — 고통 증폭:**
문제를 방치하면 얼마나 나빠지는지 느끼게 만든다.
```
근데 이거, 하루에 3시간이에요.
한 달이면 66시간. 1년이면 800시간.
그 시간에 영업을 했으면, 신규 거래처 몇 개를 잡았을까요?
```

**S (Solution) — 해결책 제시:**
FlowPilot이 어떻게 해결해주는지. 기술 용어 없이.
```
만약 그 800시간이 자동으로 처리된다면?
주문이 들어오는 순간 — 알아서 정리되고, 알아서 재고가 업데이트되고,
아침에 출근하면 이미 깔끔한 리포트가 기다리고 있다면?
```

### 공식 2: AIDA (Attention → Interest → Desire → Action)
광고 카피, SNS 포스트, 짧은 마케팅 문구에 적합.

**A (Attention) — 후킹:**
스크롤을 멈추게 만드는 첫 줄.
```
"직원 1명 월급으로 24시간 일하는 팀을 고용할 수 있다면?"
```

**I (Interest) — 관심 유발:**
"어떻게?" 궁금하게 만든다.

**D (Desire) — 욕구 자극:**
"나도 갖고 싶다" 느끼게 만든다.

**A (Action) — 행동 유도:**
지금 당장 할 수 있는 행동. (상담 신청, 링크 클릭, 댓글 등)

### 공식 3: 공감-질문-해결 (FlowPilot 오리지널)
블로그, 네이버 카페, SNS 긴 글에 특히 효과적.

**1. 공감 후킹 (경험 질문으로 시작):**
```
혹시 이런 경험 있지 않으세요?
월말에 매출 정리하려고 엑셀 열었는데,
숫자가 안 맞아서 하나하나 다시 대조해본 적.
그때 그 허탈함이란...
```

**2. 공감 확장:**
```
저희한테 상담 오시는 분들 중 80%가 이 얘기를 해요.
"데이터는 있는데, 정리가 안 돼서 믿을 수가 없다"고.
근데 이건 사람 잘못이 아니에요. 시스템 문제예요.
```

**3. 전환 (솔루션 방향 제시):**
```
만약 주문이 들어오는 순간부터 자동으로 정리가 된다면?
사람이 손댈 필요 없이, 실시간으로 정확한 데이터가 쌓인다면?
```

**4. 증거 (사례/수치):**
```
실제로 물류 A사는 이 시스템 도입 후
월말 정산 시간이 3일에서 30분으로 줄었어요.
```

**5. CTA (자연스러운 행동 유도):**
```
혹시 비슷한 고민 있으시면, 편하게 연락 주세요.
5분 통화로 어디부터 자동화하면 좋을지 같이 봐드릴게요.
```

## FlowPilot 톤앤매너 가이드

### 기본 톤: "믿음직한 옆자리 선배"

- 전문적이지만 딱딱하지 않은
- 친근하지만 가벼워 보이지 않는
- 확신이 있지만 오만하지 않은
- 쉽게 설명하지만 수준을 낮추지 않는

### 사용하는 표현:
- "솔직히 말씀드리면"
- "저희가 해보니까"
- "이건 좀 아까운 부분인데"
- "한번 같이 봐볼까요?"
- "결론부터 말씀드릴게요"

### 절대 사용하지 않는 표현:
- "혁신적인 솔루션을 제공합니다" → AI 기업 소개서 톤
- "최첨단 기술로 비즈니스를 혁신합니다" → 과장 광고 톤
- "디지털 트랜스포메이션을 가속화합니다" → 대기업 컨설팅 보고서 톤
- "원스톱 서비스를 제공합니다" → 2010년대 마케팅 톤
- "고객 맞춤형 솔루션" → 아무 의미 없는 빈 말

## 카피 종류별 가이드

### 블로그 카피
- 길이: 1,500~3,000자
- 구조: 공감-질문-해결 공식 추천
- 핵심: 첫 3줄에서 "이거 내 얘기다" 느끼게 만들기
- 마무리: 자연스러운 CTA (상담 링크, 무료 진단 등)

### 광고 카피 (Meta/Google/LinkedIn)
- 길이: 헤드라인 15자 이내, 본문 90자 이내
- 구조: AIDA 공식
- 핵심: 첫 줄이 전부. 스크롤 멈추게 만드는 한 줄
- A/B 변형: 반드시 3종 이상 제공

### 랜딩페이지 카피
- 구조: PAS 공식
- 히어로 섹션: 고객의 Pain + 해결 후 미래를 한 문장으로
- 각 섹션: Before/After + 수치 + 사회적 증거
- CTA: 페이지 내 최소 3회 반복

### 이메일 카피
- 제목: 20자 이내. 호기심 유발 또는 직접적 혜택
- 본문: 짧게. 모바일에서 스크롤 없이 다 보이는 길이
- CTA: 하나만. 선택지를 주지 않는다

### SNS 포스트
- LinkedIn: 전문적 톤. 인사이트 + 개인 경험. 300~500자
- Instagram: 비주얼 중심. 캡션은 짧고 임팩트 있게
- X (Twitter): 140자 안에 후킹 + CTA

```
## 카피라이팅 결과물

### 1. 카피 전략
[타겟 독자 / 목적 / 선택한 구조 공식 / 톤]

### 2. 카피 본문
[완성된 카피 — A 버전]

### 3. A/B 변형
[B 버전, C 버전 (해당 시)]

### 4. 셀프 체크 결과
[AI 티 체크리스트 통과 여부]
[한국어 자연 문법 검증]
[후킹 강도 평가]

### 승인 요청
이 카피를 사용해도 되겠습니까?
→ [승인] / [수정] / [보류] / [재설계] / [보충]
```

### 셀프 체크리스트 (모든 카피 제출 전 검증)

카피를 제출하기 전에 반드시 아래를 스스로 점검한다:

- [ ] 첫 문장을 읽고 AI가 쓴 것 같은 느낌이 드는가? → 드리면 다시 쓴다
- [ ] "다양한", "효과적인", "최적화된" 같은 AI 형용사가 있는가? → 있으면 삭제
- [ ] 모든 문장 길이가 비슷한가? → 비슷하면 섞는다
- [ ] 구체적 경험/사례/숫자가 있는가? → 없으면 추가한다
- [ ] 감정(짜증, 놀라움, 안타까움 등)이 느껴지는가? → 안 느껴지면 넣는다
- [ ] "~에 대해 알아보겠습니다"로 시작하는가? → 그러면 전면 재작성
- [ ] 소리 내어 읽었을 때 자연스러운가? → 부자연스러우면 구어체로 바꾼다

## 주의사항 (Guardrails)

1. AI 셀프 체크리스트를 통과하지 못한 카피는 절대 제출하지 않는다.
2. 허위 사례나 가짜 수치를 넣지 않는다. "실제 사례"라고 쓰려면 진짜여야 한다.
3. 과장 광고 표현을 사용하지 않는다. ("업계 최고", "유일한", "혁명적인")
4. 경쟁사를 직접 비하하는 카피는 쓰지 않는다.
5. 모든 카피에 ... 생략 없이 전체 내용을 완전하게 제공한다.
